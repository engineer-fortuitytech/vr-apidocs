Appetite Check API

Integrate appetite intelligence into your apps.

The Appetite Check API lets you capture meal intents, validate orders, and power smart menu flows in your own applications. Use it from mobile, web, or backend services with a simple REST interface.

How to use the API

  1. 1
    Get an API key

    Obtain your API key from the Velocity Risk Partner team and store it securely. You’ll send it as a header on every request: 

    x-api-key: <YOUR_API_KEY>
  2. 2
    Call endpoints from your app

    Use the REST endpoints to submit appetite checks, retrieve recommendations, and manage configurations. The API speaks standard JSON over HTTPS.

    Order of Operations.
    Start by sending a POST request to the /appetite-check endpoint with user appetite data:

  3. 3
    Handle responses gracefully

    Responses return machine-friendly JSON plus HTTP status codes. You can plug results directly into UI components like “recommended dishes” or “menu sections”.

Key features

  • Structured appetite checks

    Capture contextual info like time of day, dietary preferences, and hunger levels.

  • Smart recommendations

    Use the API’s scoring logic to surface suggested dishes or menu flows tailored to each user.

  • Operational monitoring

    A built-in health check lets you quickly verify API availability from your dashboards or uptime monitors.

Typical use cases

  • Restaurant apps

    Suggest dishes based on appetite level, diet type, and previous orders.

  • Nutrition planners

    Combine appetite signals with macro targets to build daily or weekly meal plans.

  • Chatbots / assistants

    Let conversational agents call the API to decide when and what to suggest for the next meal.
